# Тема проекта
> Разработка Telegram Mini App как платформа для следующих функций:
1) Ежедневный гороскоп - персонализированные астрологические прогнозы с использованием внешнего API
2) Карта дня - генерация карты Таро с детальным толкованием с использованием внешнего API
3) Тест на определение типа - интерактивный опрос "Темный или светлый маг?"
# Определение проблемы
Основная проблема современных пользователей, интересующихся самопознанием и эзотерикой, заключается в высокой фрагментации качественных сервисов и информационном хаосе, из-за чего получение ежедневных астрологических прогнозов или работа с картами Таро превращается в рутинный поиск по разрозненным ресурсам. Отсутствие единой, локализованной и технически доступной платформы внутри привычных средств коммуникации (мессенджеров) не только создает барьер, но и значительно снижает концентрацию на процессе самоанализа, лишая пользователя возможности систематизировать свои практики и удобно отслеживать динамику личного прогресса.
# Выработка требований
## Общие требования: 
> Приложение – Telegram mini App, через которое можно посмотреть различные формы предсказания на день.
## Требование к получению гороскопа:
> Когда я хочу понять, что меня ждет сегодня, я хочу иметь возможность узнать по своему гороскопу предсказание, чтобы спланировать свой день с его учетом.

## Требование к получению карты дня:
> Когда мне нужен совет или предсказание, я хочу получить карту дня, чтобы понять как мне действовать в этот день и чего ожидать.

## Требование к прохождению теста:
> Когда я хочу развлечься и узнать о себе, я хочу пройти тест на определение типа мага, чтобы понять свою магическую принадлежность

## Требование к книгам о магии:
> Когда я хочу углубить свои познания о магии, я хочу получить список книг, чтобы самостоятельно изучить эзотерику.

# Архитектура системы
## FRONTEND
> Реализация пользовательского интерфейса в формате приложения. Фронтенд-приложение разработано с использованием языков html, css, js. Связано с Vercel и подтягивает обработку логики из Backend.

## BACKEND
> Cерверная часть приложения, реализованная с использованием Render на языке Python, также подключает telegram бота.

## База данных
> Хранит данные, передающееся пользователю. Связана с SupaBase.
